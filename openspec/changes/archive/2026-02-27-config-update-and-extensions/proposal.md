## Why

Конфигурация 1С — живая система, которая регулярно обновляется. Сейчас после каждого изменения конфигурации нужно вручную экспортировать файлы и перезапускать контейнеры с `RESET_DATABASE=true`, что приводит к многочасовой полной переиндексации. Нет удобного механизма обновления. Кроме того, помимо основной конфигурации используются расширения (extensions), чьи метаданные и код сейчас не индексируются MCP-серверами.

## What Changes

- Создать скрипт/процедуру обновления индексов MCP-серверов при изменении конфигурации 1С, минимизирующую простой и время переиндексации
- Исследовать и реализовать способ индексации расширений конфигурации (выгрузка расширений в файлы → подключение к MCP-серверам через volume mounts)
- Обновить `.env.example` и документацию с описанием процедуры обновления и работы с расширениями

## Capabilities

### New Capabilities
- `config-reindex`: Механизм обновления индексов MCP-серверов при изменении конфигурации — скрипт/процедура с выбором режима (полная/частичная переиндексация), контролем параметров RESET_DATABASE/RESET_CACHE, и валидацией результата
- `extensions-indexing`: Поддержка индексации расширений конфигурации 1С — настройка дополнительных путей для volume mounts, документация по экспорту расширений, проверка что серверы корректно индексируют данные расширений

## Impact

- **Скрипты**: новый `mcp-deployment/scripts/reindex.sh` для управления переиндексацией
- **Конфигурация**: новые переменные в `.env` для путей к расширениям
- **Docker Compose**: дополнительные volume mounts для каталогов расширений
- **Документация**: CLAUDE.md, AGENTS.md — процедура обновления и работа с расширениями
- **Сервер alcor**: тестирование на реальных данных
