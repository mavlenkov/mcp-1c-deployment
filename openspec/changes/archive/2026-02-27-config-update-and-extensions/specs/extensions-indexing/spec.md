## ADDED Requirements

### Requirement: Переменная окружения для пути к расширениям
`.env.example` ДОЛЖЕН содержать переменную `EXTENSIONS_PATH` для указания каталога с экспортированными расширениями конфигурации 1С.

#### Scenario: Настройка пути к расширениям
- **WHEN** пользователь указывает `EXTENSIONS_PATH=/home/l7777/mcp/extensions` в `.env`
- **THEN** серверы CodeMetadata/CloudEmbeddings/Graph монтируют этот каталог и индексируют его содержимое

#### Scenario: Расширения не настроены
- **WHEN** `EXTENSIONS_PATH` не указан или пуст
- **THEN** серверы работают как прежде — индексируют только основную конфигурацию

### Requirement: Volume mounts для расширений в docker-compose
Docker-compose файлы ДОЛЖНЫ монтировать каталог расширений как дополнительный volume для серверов, работающих с кодом конфигурации.

#### Scenario: CodeMetadataSearchServer с расширениями
- **WHEN** `EXTENSIONS_PATH` задан и docker-compose запущен
- **THEN** каталог расширений примонтирован в контейнер CodeMetadata и его содержимое индексируется вместе с основной конфигурацией

#### Scenario: CloudEmbeddingsServer с расширениями
- **WHEN** используется CloudEmbeddingsServer и `EXTENSIONS_PATH` задан
- **THEN** каталог расширений примонтирован и индексируется

### Requirement: Документация по экспорту расширений
CLAUDE.md ДОЛЖЕН содержать инструкцию по экспорту расширений из 1С и их размещению для индексации MCP-серверами.

#### Scenario: Инструкция по расширениям
- **WHEN** пользователь ищет информацию о расширениях в документации
- **THEN** находит описание: как экспортировать расширение из Конфигуратора, куда положить файлы, как перенастроить MCP-серверы
