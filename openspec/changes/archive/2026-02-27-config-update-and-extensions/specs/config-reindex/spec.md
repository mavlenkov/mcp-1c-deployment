## ADDED Requirements

### Requirement: Скрипт переиндексации reindex.sh
Система ДОЛЖНА предоставлять скрипт `mcp-deployment/scripts/reindex.sh` для управления переиндексацией MCP-серверов при обновлении конфигурации 1С.

#### Scenario: Полная переиндексация всех серверов
- **WHEN** пользователь выполняет `./scripts/reindex.sh --all`
- **THEN** скрипт перезапускает все серверы с `RESET_DATABASE=true`, а затем возвращает `RESET_DATABASE=false` в `.env`

#### Scenario: Переиндексация выбранных серверов
- **WHEN** пользователь выполняет `./scripts/reindex.sh --servers metadata,graph`
- **THEN** только указанные серверы перезапускаются с `RESET_DATABASE=true`

#### Scenario: Справка по использованию
- **WHEN** пользователь выполняет `./scripts/reindex.sh` без параметров или с `--help`
- **THEN** выводится справка с описанием доступных серверов и режимов

### Requirement: Контроль целостности данных при переиндексации
Скрипт `reindex.sh` ДОЛЖЕН предупреждать пользователя о времени переиндексации и запрашивать подтверждение перед запуском.

#### Scenario: Подтверждение перед переиндексацией
- **WHEN** пользователь выполняет `./scripts/reindex.sh --all`
- **THEN** скрипт выводит предупреждение о длительности операции и запрашивает подтверждение (y/n)

#### Scenario: Пропуск подтверждения
- **WHEN** пользователь выполняет `./scripts/reindex.sh --all --yes`
- **THEN** переиндексация запускается без подтверждения

### Requirement: Документация процедуры обновления
CLAUDE.md и AGENTS.md ДОЛЖНЫ содержать описание процедуры обновления конфигурации (экспорт → копирование → reindex).

#### Scenario: Документация содержит пошаговую инструкцию
- **WHEN** пользователь ищет информацию об обновлении конфигурации в CLAUDE.md
- **THEN** находит раздел с описанием шагов: экспорт из 1С, копирование файлов, запуск reindex.sh
