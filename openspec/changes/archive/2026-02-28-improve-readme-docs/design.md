## Context

README.md уже содержит таблицу серверов, quick start и разделы по обновлению конфигурации. Проблемы: краткое описание, секция IDE показывает только 3 сервера, нет пояснения что конфиг нужно адаптировать под свои серверы.

## Goals / Non-Goals

**Goals:**
- Понятное описание проекта для нового пользователя
- Полный пример MCP-конфигурации со всеми серверами и комментариями
- Пояснение различий между IDE (путь к конфиг-файлу)

**Non-Goals:**
- Дублирование официальной документации (ссылка достаточно)
- Изменение структуры проекта

## Decisions

### 1. Полный конфиг vs минимальный пример
**Решение:** Показать полный конфиг со всеми серверами, но с комментариями какие обязательны, какие опциональны. Пользователь удалит ненужные.
**Альтернатива:** Минимальный пример + ссылка на документацию. Отклонено — пользователь может не найти полный пример.

### 2. Формат конфигурации
**Решение:** JSON (стандарт для Cursor/Claude Code). Для Claude Code показать также формат `claude_desktop_config.json` если отличается.

## Risks / Trade-offs

- **[Risk]** Конфиг устареет при добавлении серверов → **Mitigation:** Комментарий «актуальные порты см. в таблице выше»
