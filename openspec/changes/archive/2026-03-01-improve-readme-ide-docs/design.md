## Context

README.md содержит раздел "Подключение к IDE" с JSON-примером конфигурации из 8 серверов. Проблема: JSON не поддерживает комментарии, пользователь видит список URL без пояснений и не понимает, какие серверы обязательны, а какие опциональны. Ссылка на документацию есть в конце файла, но её легко пропустить.

## Goals / Non-Goals

**Goals:**
- Сделать раздел IDE-конфигурации самодостаточным: пользователь должен понять назначение каждого сервера без перехода на внешнюю документацию
- Вынести ссылку на документацию в заметное место (рядом с другими важными ссылками в шапке)
- Дать рекомендации по минимальной конфигурации

**Non-Goals:**
- Не менять структуру остальных разделов README
- Не дублировать полную документацию — только краткие описания и ссылка на подробности

## Decisions

### 1. Формат описания серверов в JSON-примере

**Решение:** Добавить таблицу с описанием каждого сервера перед JSON-примером. JSON оставить чистым (без псевдокомментариев), а описания вынести в таблицу-легенду.

**Альтернатива:** Использовать JSONC с комментариями `//` — отвергнуто, т.к. не все IDE парсят JSONC, и копирование сломает валидный JSON.

### 2. Минимальная vs полная конфигурация

**Решение:** Показать два примера: "Минимальная конфигурация" (3-4 ключевых сервера) и "Полная конфигурация" (все серверы). Это даёт понять, что не обязательно подключать всё сразу.

### 3. Размещение ссылки на документацию

**Решение:** Оставить ссылку в текущем месте (шапка уже содержит блок "> Документация"), но также добавить её в раздел IDE с пометкой "подробнее о каждом сервере".

## Risks / Trade-offs

- [Дублирование информации между таблицей серверов в начале и описаниями в IDE-разделе] → Использовать краткие описания в IDE-разделе (1 строка), ссылаться на таблицу для подробностей
- [README станет длиннее] → Минимальное увеличение (одна таблица + один дополнительный JSON-блок)
