## ADDED Requirements

### Requirement: README SHALL have expanded project description
README.md SHALL содержать развёрнутое описание: что такое MCP-серверы, как они расширяют возможности AI-ассистентов для 1С-разработки.

#### Scenario: Новый пользователь понимает назначение
- **WHEN** пользователь открывает README на GitHub
- **THEN** первые два абзаца объясняют что это за проект и зачем он нужен

### Requirement: Ссылка на документацию
README SHALL содержать ссылку на официальную документацию (https://docs.onerpa.ru/mcp-servery-1c) в разделе "Подключение к IDE" с пометкой о подробных инструкциях по каждому серверу.

#### Scenario: Пользователь находит ссылку на документацию в контексте IDE
- **WHEN** пользователь читает раздел подключения к IDE
- **THEN** он видит ссылку на подробную документацию с пояснением, что там описаны возможности каждого сервера

### Requirement: IDE config SHALL include all servers
Секция подключения к IDE SHALL содержать полный конфигурационный файл со всеми 9 серверами, с комментариями о назначении каждого и пояснением что можно подключить только нужные.

#### Scenario: Пользователь видит полный конфиг
- **WHEN** пользователь читает секцию «Подключение к IDE»
- **THEN** конфиг содержит все серверы с портами и пояснениями

### Requirement: IDE config SHALL explain file location per IDE
Секция SHALL пояснять где размещать конфигурационный файл для разных IDE (Cursor, Claude Code).

#### Scenario: Пользователь Cursor знает куда положить файл
- **WHEN** пользователь использует Cursor
- **THEN** README объясняет путь к файлу `.cursor/mcp.json`

#### Scenario: Пользователь Claude Code знает куда положить файл
- **WHEN** пользователь использует Claude Code
- **THEN** README объясняет путь к файлу `.claude.json` или конфигурацию через настройки
